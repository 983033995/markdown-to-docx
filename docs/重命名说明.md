# 项目重命名说明

## 重命名概述

项目已从 `md2docx` 重命名为 `mdconv`（Markdown Converter），以更准确地反映其多格式转换功能。

## 新名称

**mdconv** - Markdown 多格式转换工具

### 命名理由

1. **准确性**: 项目支持多种输出格式（DOCX、PDF、HTML、PPTX、EPUB），不仅限于 DOCX
2. **简洁性**: `mdconv` 简短易记，符合 Unix 工具命名习惯
3. **专业性**: 体现了 Markdown Converter 的核心功能

## 变更内容

### CLI 命令

**新命令**: `mdconv`

```bash
# 基本使用
mdconv document.md                  # 默认转换为 DOCX
mdconv -f pdf document.md           # 转换为 PDF
mdconv -f html document.md          # 转换为 HTML
```

**向后兼容**: `md2docx` 仍然可用（作为 `mdconv` 的符号链接）

```bash
# 旧命令仍然有效
md2docx document.md
```

### 配置文件

| 旧名称 | 新名称 |
|--------|--------|
| `.md2docx.conf` | `.mdconv.conf` |
| `.md2docx.yaml` | `.mdconv.yaml` |
| `.md2docx_last.yaml` | `.mdconv_last.yaml` |

### 文件结构

```
markdown-to-docx/
├── bin/
│   ├── mdconv          # 主命令（新）
│   └── md2docx         # 符号链接（向后兼容）
├── .mdconv.conf        # 配置文件（新）
└── ...
```

## 支持的格式

### 输出格式

| 格式 | 说明 | 命令示例 |
|------|------|----------|
| DOCX | Microsoft Word | `mdconv -f docx file.md` |
| PDF | 便携文档格式 | `mdconv -f pdf file.md` |
| HTML | 网页格式 | `mdconv -f html file.md` |
| PPTX | PowerPoint | `mdconv -f pptx file.md` |
| EPUB | 电子书 | `mdconv -f epub file.md` |

### PDF 引擎

| 引擎 | 特点 | 安装 |
|------|------|------|
| WeasyPrint | 轻量级（20MB） | `brew install weasyprint` |
| Chromium | 现代化渲染（200MB） | `brew install --cask chromium` |
| XeLaTeX | 专业排版（4GB） | `brew install --cask mactex-no-gui` |

## 迁移指南

### 对现有用户

**无需任何操作！**

- ✅ 旧命令 `md2docx` 仍然有效
- ✅ 旧配置文件会自动兼容
- ✅ 所有功能保持不变

### 推荐更新

虽然不是必需的，但建议逐步迁移到新命令：

1. **更新命令行使用**:
   ```bash
   # 旧方式
   md2docx document.md
   
   # 新方式（推荐）
   mdconv document.md
   ```

2. **更新脚本引用**:
   如果您的脚本中使用了 `md2docx`，建议更新为 `mdconv`

3. **重新运行安装**:
   ```bash
   ./install.sh
   ```
   这将更新配置文件和 PATH 设置

## 常见问题

### Q: 我需要重新安装吗？

A: 不需要。旧命令仍然有效。但建议运行 `./install.sh` 以获得最新功能。

### Q: 我的旧配置文件怎么办？

A: 旧配置文件会继续工作。新安装会创建新的配置文件。

### Q: md2docx 命令会被移除吗？

A: 不会。我们会长期保持向后兼容性。

### Q: 如何查看所有支持的格式？

A: 运行 `mdconv --help` 或查看 README.md

## 技术细节

### 符号链接

`md2docx` 是 `mdconv` 的符号链接：

```bash
$ ls -l bin/
mdconv -> mdconv
md2docx -> mdconv
```

### 配置文件优先级

1. `~/.mdconv.yaml` (用户配置)
2. `~/.mdconv.conf` (系统配置)
3. 默认配置
